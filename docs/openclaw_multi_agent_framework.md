# OpenClaw 多 Agent 框架落地说明

## 目标

在现有 `OpenClaw` 和 `ai-trading-research` 的基础上，建立一套适合 `美股 + A股市场分析` 与 `自媒体生产` 的轻量多 Agent 生产线。

这套方案优先解决 3 个问题：

- 把 `研究` 和 `写作` 分开，避免一轮对话里同时做事实搜集、判断和包装
- 把 `快讯` 和 `深度内容` 分开，保证日更效率与长文质量
- 把 `观点生成` 和 `质量把关` 分开，降低错引事实和表述过度确定的风险

## 与现有项目的衔接

当前项目已经提供了适合 `OpenClaw` 读取的 A 股日任务产物，见 `docs/openclaw_daily.md`。

其中 A 股分析可直接复用以下输出：

- `reports/openclaw_daily/latest/manifest.json`
- `reports/openclaw_daily/latest/daily_candidates.csv`
- `reports/openclaw_daily/latest/daily_report.md`
- `reports/openclaw_daily/latest/daily_signals.json`

建议把这些文件视为 `市场扫描员` 的结构化输入源。

对于美股，建议先保持同一套 Agent 角色不变，仅替换输入源：

- 财报日历
- 宏观事件日历
- 新闻摘要
- 重点股票池或主题池
- 盘前/盘后异动列表

## 第一版推荐采用 5 Agent 轻量版

### `总控主编`

职责：

- 接收用户任务
- 判断是 `日更快讯` 还是 `深度研究`
- 分派给下游 Agent
- 汇总多方结果并输出最终结论
- 在未启用独立审校 Agent 时承担最终把关

适合模型：

- 主模型：`aicoding-2233/claude-opus-4-6`
- 备用：`aicoding-2233-responses/gpt-5.3-codex`

### `市场扫描员`

职责：

- 生成盘前、盘中、盘后重点观察清单
- 汇总板块、指数、热点资产和事件日历
- 从结构化文件中提炼出高信号密度摘要

适合模型：

- 主模型：`aicoding-2233-responses/gpt-5.3-codex`
- 备用：`moonshot/kimi-k2.5`

### `基本面研究员`

职责：

- 处理行业、公司、政策、财报、估值与催化
- 输出研究底稿，不直接做最终成文
- 对 `A股` 与 `美股` 使用一致的研究模板

适合模型：

- 主模型：`moonshot/kimi-k2.5`
- 备用：`aicoding-2233/claude-opus-4-6`

### `新闻情绪研究员`

职责：

- 归纳新闻、公告、纪要、社媒与市场叙事
- 输出情绪温度、共识点、分歧点和潜在误读
- 单独标注消息来源强弱与时效性

适合模型：

- 主模型：`moonshot/kimi-k2.5`
- 备用：`aicoding-2233-responses/gpt-5.3-codex`

### `内容写手`

职责：

- 把研究结论改写成快讯、短评、长文或视频脚本
- 针对不同平台做语气和篇幅切换
- 不新增未经验证的事实

适合模型：

- 主模型：`moonshot/kimi-k2.5`
- 备用：`aicoding-2233/claude-opus-4-6`

## 模型路由原则

### `Claude Opus 4.6`

优先用于：

- 任务拆解
- 多 Agent 结果合并
- 长上下文整合
- 最终定稿

### `GPT 5.3 Codex`

优先用于：

- 结构化摘要
- 固定模板输出
- 表格/JSON/清单整理
- 市场扫描结果标准化

### `Kimi`

优先用于：

- 中文信息归集
- 新闻与舆情整理
- 中文内容起草
- 行业与公司资料摘要

## 两个运行模式

### 模式 1：日更快讯

触发场景：

- 盘前要点
- 午盘简报
- 盘后总结
- 突发事件快评

默认协作链：

`总控主编 -> 市场扫描员 -> 新闻情绪研究员 -> 总控主编 -> 内容写手 -> 总控主编`

输出内容：

- 100 到 200 字快讯
- 300 到 600 字观点短评
- 60 到 120 秒口播稿

### 模式 2：深度研究

触发场景：

- 行业专题
- 个股深挖
- 政策影响拆解
- 财报/估值/预期差分析

默认协作链：

`总控主编 -> 基本面研究员 + 新闻情绪研究员 + 市场扫描员 -> 总控主编 -> 内容写手 -> 总控主编`

输出内容：

- 研究提纲
- 长文初稿
- 视频脚本
- 可拆条的社媒素材

## 内容分层

建议所有 Agent 最终围绕统一的 4 层内容产出：

- `L1`：快讯，100 到 200 字
- `L2`：短评，300 到 600 字
- `L3`：长文，1000 到 3000 字
- `L4`：视频脚本，含开头钩子、主体分段和结尾 CTA

## 质量控制

第一版不强制拆出独立 `事实核查员` 和 `合规编辑`，但要求 `总控主编` 在终稿前完成以下检查：

- 数字、日期、公司名、代码是否一致
- 事实、解释、预测是否分开表达
- 是否把推测写成了确定性结论
- 是否缺少风险点、反例或失效条件

当日更流程稳定后，再扩展为 8 Agent 版：

- `技术面分析师`
- `多空辩论员`
- `事实核查员`
- `合规编辑`

## 文件说明

本次落地文件包括：

- `docs/openclaw_multi_agent_framework.md`
- `openclaw/agent_roles.yaml`
- `openclaw/handoff_schema.yaml`
- `openclaw/workflows.yaml`
- `openclaw/prompts/*.md`

推荐使用顺序：

1. 先阅读本文件确认角色边界
2. 再看 `openclaw/agent_roles.yaml` 了解模型分工
3. 再看 `openclaw/handoff_schema.yaml` 统一交接字段
4. 最后把 `openclaw/prompts/*.md` 复制到你的 OpenClaw Agent 提示词中
